CVM++ Project Report
====================

1. Introduction
---------------
CVM++ is a high-performance, stack-based virtual machine and compiler developed entirely from scratch in C++17. It implements a fully functioning programming language and toolchain, providing a deep dive into compiler construction and low-level bytecode execution environments. The project is designed with strict runtime safety, optimization passes, and memory efficiency in mind.

2. Language Features
--------------------
The CVM++ language provides a robust feature set expected in modern structured programming:
- Deep Block Scoping: Full lexical scoping with variable shadowing. Scopes are automatically cleaned up upon exit.
- Control Flow: Support for `if/else`, `while`, and `for` loops, including `break` and `continue` constructs.
- Short-Circuit Evaluation: Logical operators (`&&`, `||`) intelligently skip evaluation of the right-hand operand when the outcome is already known.
- Rich Operators: Comprehensive set of arithmetic, unary, logical, and bitwise operators.
- Numeric Literals: Support for decimal, hexadecimal, binary, and octal integer literals.

3. System Architecture
----------------------
The CVM++ architecture is divided into five distinct stages, forming a complete compilation and execution pipeline:

a) Lexer: Tokenizes the raw source code. It handles comments, numeric bases, and keyword matching.
b) Parser: A recursive-descent parser that consumes tokens and constructs an Abstract Syntax Tree (AST). It uses a lightweight `NodeType` enum instead of RTTI (`dynamic_cast`), ensuring zero-overhead type identification.
c) Compiler: A single-pass AST visitor that directly emits bytecode. It performs compile-time constant folding on binary, unary, and logical expressions to eliminate runtime overhead.
d) Bytecode Format: Instructions are tightly packed into a `Chunk`. A crucial optimization is the use of Run-Length Encoding (RLE) to map bytecode offsets back to source line numbers, drastically reducing the memory footprint.
e) Virtual Machine: A stack-based execution engine that runs the compiled bytecode. It enforces strict boundary checks to prevent stack overflow/underflow and uses explicit sign-bit replication for safe arithmetic right shifts on negative integers.

4. Developer Tools
------------------
CVM++ includes tools to assist in debugging and execution:
- Interactive REPL: An environment to execute code line-by-line with robust error recovery.
- Bytecode Disassembler: An integrated disassembler (`--dump` flag or `disasm on` REPL command) that outputs human-readable representations of the generated bytecode, absolute jump targets, and operands.

5. Testing and Validation
-------------------------
The reliability of the system is ensured via a comprehensive suite of regression and feature tests (located in the `tests/` directory). Tests validate everything from complex nested loops and block scoping to edge cases like `INT32_MIN` representations and constant folding correctness.

6. Conclusion
-------------
CVM++ successfully demonstrates a complete, optimized language toolchain. Through careful design decisions such as RLE memory mappings, constant folding, and strict runtime protections, it achieves an excellent balance of speed, safety, and maintainability.
